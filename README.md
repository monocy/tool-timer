# Premium Timer (タイマーツール)

Free Web Tools の **Premium Timer (タイマーツール)** リポジトリです。
このツールは、Free Web Tools 全体のツール開発において、共通 of システム設計・開発方法を具現化した **標準的な構成** になっています。

---

## 1. 設計思想

このプロジェクトでは、全ツールが同一のユーザー体験、カラーテーマ（ダーク・ライト・ネオン）、設定モーダル（言語切替・フォントサイズ）、背景画像管理を共有できるように、**コア機能とレイアウトの共通化（プラットフォーム化）** を行っています。

### 構成
* **コアロジック・スタイル（親ポータルから共通提供）**:
  * **共通CSS (`static/css/portal.css`)**: テーマカラー変数（`:root` 等）、スクロールバー、およびガラスモーフィズム（`.glass-panel`）等の共通スタイル定義。
  * **共通JS (`static/js/portal_common.js`)**: 設定情報（LocalStorage 保存）、多言語切替ロジック、文字サイズ変更処理などを提供する Compositions 共通ロジック `usePortalCommon`。
  * **共通コンポーネント (`static/js/portal_components.js`)**: ポータルと統一されたヘッダー（`PortalHeader`）、フッター（`PortalFooter`）、および右下全画面切り替えボタン（`PortalControls`）。
* **ツール独自ロジック (このリポジトリ内)**:
  * ツール独自の機能などはすべて `js/workspace.js`（`ToolWorkspace` コンポーネント）内に隔離して記述されます。これにより、ツール自体の開発に集中できます。

---

## 2. 開発方法 (dev.html でのスタンドアロン開発)

ローカル環境での新規ツールの作成、デバッグ、挙動の確認は **`dev.html`** をブラウザで直接開いて行います。

### 利点
* **CORS制限のバイパス**:
  ローカルの HTML ファイルをブラウザで直接ダブルクリックで開いた場合（`file:///` プロトコル）、セキュリティ上の制限（CORS）によって同一フォルダ内以外のファイル読み込みや一部通信がブロックされます。
  `dev.html` は外部の共通 JS/CSS に依存せず、最小限のインラインスタイル、内蔵フォールバック翻訳、ローカルの `i18n.json` からの直接 fetch（失敗時は内蔵データを自動使用）によって、**一切のローカル Web サーバーを起動することなく動作検証が完了**します。
* **高速な開発サイクル**:
  `js/workspace.js` をエディターで書き換え、`dev.html` をブラウザでリロードするだけで動作確認ができます。

---

## 3. 各ファイルの役割と仕様

### ① `index.html` (ポータル統合用 HTML)
* **環境自動判定型アセットローダー (CORS 回避)**:
  ローカルのファイルシステム上で開かれた場合（`file:`）は 5階層上（`../../../../../static/`）の共通アセットを、サーバー経由（`http:` / `https:`）で開かれた場合は 2階層上（`../../static/`）を自動的に判別し、順にアセットを動的ロードします。
* **マウント制御**:
  動的なスクリプト読み込みがすべて完了した時点で `common-assets-loaded` イベントを発火し、その後 Vue アプリケーションを安全に初期化・マウントします。
* **タイトルフォールバックロジック**:
  CORS制限によって `i18n.json` が取得できない場合、自動的に HTML 内に定義されたオリジナルの静的タイトル（`<title>`）を維持して表示し、タイトル名が `page_title` といったキー名に化けるのを防ぎます。

### ② `js/workspace.js` (ツール本体)
* ツール自体のマークアップおよび Vue.js ロジックを実装します。
* 親から `settings` および `translations` プロパティを受け取り、共通の翻訳ヘルパー `t(key)` を用いて多言語化します。

### ③ `i18n.json` (多言語定義ファイル)
* 英語（`en`）および日本語（`ja`）の翻訳定義ファイルです。
* `page_title` (ブラウザタブ of タイトル)、`tool_title` (ヘッダー用タイトル) などの規定キーを定義します。

---

## 4. 新規ツールを追加する際の手順

1. `tool-template/` ディレクトリをコピーして、新しいツールディレクトリ（例: `tools/my-new-tool`）を作成します。
2. `index.html` および `dev.html` 内の `<title>` タグを任意のツール名に変更します。
3. `i18n.json` 内の翻訳定義を書き換えます。
4. `js/workspace.js` の `ToolWorkspace` コンポーネントを編集して、ツールのロジックとデザインを実装します（`dev.html` で動作確認を行います）。
5. 開発完了後、親ポータルの管理 JSON やルーティングに登録することで、共通テーマと設定を引き継いだ形でポータル上に統合されます。
